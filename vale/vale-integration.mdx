## Overview

vale checks `.md` and `.mdx` files for prose quality before each commit. It
enforces the Uwazi style guide automatically. Error-level alerts block the
commit. Warnings are visible but non-blocking.

`rumdl` runs alongside vale and owns Markdown structure and layout. It replaces
`ESLint` and Prettier on `.md` and `.mdx` files. Its configuration lives in
`.rumdl.toml` at the repo root.

## Prerequisites

Complete these steps before the lint chain runs on your machine.

1. Install vale. Follow the instructions for your operating system at
   [vale.sh](https://vale.sh/docs/vale-cli/installation/).

2. Download the third-party style packages:

   ```bash
   vale sync
   ```

3. Install `npm` dependencies (includes `mdx2vast` for MDX support):

   ```bash
   npm install
   ```

4. Install `rumdl`:

   ```bash
   brew install rumdl
   ```

## How it works

vale runs as part of `lint-staged` on every commit. The wrapper script
`vale/scripts/vale-check.sh` runs vale twice:

- First pass: shows all warnings and errors for visibility.
- Second pass: checks for errors only. A non-zero exit code blocks the commit.

`rumdl` runs after vale on the same staged files. `ESLint` and Prettier no
longer touch Markdown, so structural and layout concerns now belong to `rumdl`.

If vale isn't installed, the pre-commit hook skips prose checking and shows a
warning. Contributors without vale can still commit, but nothing catches prose
errors until they install vale.

## Running vale manually

To check the full `docs/` directory:

```bash
vale --minAlertLevel warning docs/
```

To check a single file:

```bash
vale --minAlertLevel warning docs/how-to/my-guide.mdx
```

## Rules reference

| Rule                        | Level   | Checks                                         |
| --------------------------- | ------- | ---------------------------------------------- |
| `Uwazi.UwaziCase`           | error   | Lowercase `uwazi` in prose                     |
| `Uwazi.UIVerbs`             | error   | Banned UI verbs (`click`, `tap`)               |
| `Uwazi.LinkText`            | error   | Vague link text (`here`, `link`, `read more`)  |
| `Uwazi.Spelling`            | error   | Spelling against Oxford English dictionary     |
| `Uwazi.ImageAltText`        | error   | Missing or generic image alt text              |
| `Uwazi.OxfordComma`         | error   | Missing Oxford comma in lists                  |
| `Uwazi.Contractions`        | warning | Expanded forms that should be contracted       |
| `Uwazi.FleschReadingEase`   | warning | Score below 70                                 |
| `Uwazi.GunningFog`          | warning | Score above 10                                 |
| `Uwazi.HeadingPunctuation`  | warning | End punctuation in headings                    |
| `Uwazi.HeadingSentenceCase` | warning | Title-case headings                            |
| `Uwazi.Idioms`              | warning | Idioms and business cliches                    |
| `Uwazi.InclusiveLanguage`   | warning | Non-inclusive terms                            |
| `Uwazi.OrdinalNumbers`      | warning | Ordinal numerals (`1st`, `2nd`) in prose       |
| `Uwazi.ParagraphLength`     | warning | Paragraphs longer than 5 sentences             |
| `Uwazi.Passive`             | warning | Passive voice                                  |
| `Uwazi.RespectfulTone`      | warning | Condescending or minimizing language           |
| `Uwazi.SentenceLength`      | warning | Sentences longer than 25 words                 |
| `Uwazi.SMOG`                | warning | SMOG grade above 10                            |
| `Uwazi.SpelledOutNumbers`   | warning | Spelled numbers 10+ or numerals 1–9 in prose   |
| `Uwazi.Weasel`              | warning | Hedge words (`clearly`, `obviously`, `simply`) |
| `Uwazi.WordSubstitutions`   | warning | Complex words with simpler alternatives        |

## Not lint-checked

vale can't reliably detect the following style-guide rules. Reviewers enforce
them by eye:

- Lists: 5–7 item cap, intro text required, max 2-level nesting.
- Code blocks: language tag missing, explanation required after block.
- Call-outs: max 3 per file, max 2 sentences each.
- Highlighting: limit 2–3 bold/italic per page.
- Tables: intro text required, ≤5 columns.
- Headings: no H1 in document body (the frontmatter `title` is the H1).
- Periods: one space (not two) after period.

## Adding a rule

1. Create `vale/styles/Uwazi/RuleName.yml` with the rule definition.
2. Create `vale/tests/fixtures/RuleName/` with `.vale.ini`, `invalid.md`, and
   `valid.md`.
3. Run `npm run vale:test` to confirm the rule fires correctly.

## Testing rules

```bash
npm run vale:test
```

To test a single rule:

```bash
bash vale/scripts/test-rules.sh RuleName
```
