This knowledge hub organizes documentation using the **Diátaxis framework**,
which structures content into four distinct types based on user needs:

## 🎓 [Tutorials](/docs/tutorials) {/* #-tutorials */}

**Learning-oriented** • Step-by-step lessons for beginners

Perfect for newcomers who want to learn by doing.
Our tutorials guide you through practical exercises to build understanding from
the ground up.

## 🔧 [How-to guides](/docs/how-to) {/* #-how-to-guides */}

**Problem-oriented** • Solutions for specific tasks

Practical guides for experienced users who need to achieve specific goals
or solve particular problems.

## 📚 [Reference](/docs/reference) {/* #-reference */}

**Information-oriented** • Comprehensive technical details

Complete technical specifications, API documentation,
and detailed feature descriptions for quick lookup.

## 💡 [Explanations](/docs/explanations) {/* #-explanations */}

**Understanding-oriented** • Concepts and documentation insights

In-depth explanations of concepts, design decisions,
and documentation principles to deepen your understanding.
